#### 초기 세팅 방법
```
npm i -g yo generator-office
yo office  # Word Task Pane + React/TS 생성
```

#### 프로젝트 구성 기반
1. Taskpane에서 '/React/App.tsx'를 불러오는 것으로 시작점
2. '/React'내부에서 API를 확인 및 테스트할 수 있는 방향성대로 세부 디렉터리를 나눌 예정
3. 코드의 재사용성보다는 API의 특징을 알기 위함이 목적임. 디렉터리별로 각각 필요한 것을 따로 담았으면 좋겠음.

#### 목표
1. 한단계씩 구현을 해보며, Office.js의 API 구조를 익히기 위함
2. 구조 별로 코드에 분리를 해두거나, add-in 내부에서 섹션을 나누고자 함. (개별적으로 동작을 확인하고 분석하기 위함)
3. 구현 순서
    - 문서 최상위/실행 컨텍스트
    - 본문/범위 기반 편집 (문서 사용자에 의한 선택, Add-in에서 문서 내용 전체 읽기 등 가능한 모든 선택 방법)
    - 구조화된 입력/템플릿화
    - 표/리스트/하이퍼링크 등의 여러 요소 작업
    - 주석/코멘트/검토자 등의 작업 (추적 및 변경 흐름 워크플로우 추적)
    - 도형, 이미지, 캔버스 등의 시각 요소
    - 문서 메타데이터/속성/커스텀 데이터
